Job Purpose

The Data Scientist supports the development of data-driven routing, navigation, logistics, and mobility intelligence systems for Tasco Maps, VETC, MyTasco, fleet services, tolling, parking, insurance, logistics, and future automotive products.

This role converts GPS probes, trip logs, routing results, ETA errors, traffic signals, vehicle profiles, map attributes, and user behavior into measurable models and optimization insights.

The position is not limited to dashboarding. It is designed for a candidate who can gradually grow into building Vietnam-localized routing intelligence: time-dependent routing, route quality evaluation, driver preference modeling, logistics assignment, and AI-assisted detection of routing / map defects.

Key Responsibilities

A. Routing & ETA Data Science
  • Analyze routing logs, trip trajectories, GPS probes, route alternatives, actual-vs-planned paths, and ETA prediction errors.
  • Build features for ETA models using road class, time of day, congestion level, toll road usage, intersection complexity, turn penalty, speed profile, event signals, and vehicle type.
  • Evaluate route quality using actual travel time, distance, delay, detour ratio, rerouting frequency, user cancellation, toll cost, violation risk, and driver adherence.
  • Build dashboards and automated reports for ETA error, routing failure, road-segment speed anomaly, and map-data regression.
  • Work with Engineering to improve dynamic costing inputs for routing engines such as Valhalla, OSRM, or GraphHopper.
B. Vehicle Routing & Logistics Optimization
  • Support modeling of logistics and fleet routing problems: vehicle routing, pickup-and-delivery, capacity constraints, time windows, heterogeneous vehicle types, driver working hours, depot constraints, and service time.
  • Prototype optimization models using Python and open-source solvers for small to medium-sized planning problems.
  • Compare exact optimization, heuristic, local search, and rule-based approaches for real-world dispatching and routing use cases.
  • Build benchmark datasets for Vietnamese use cases: toll-road logistics, car-service pickup/dropoff, roadside assistance, EV charging support, vehicle inspection routing, and last-mile delivery.
C. Time-dependent Travel Time & Traffic Intelligence
  • Build time-dependent travel-time profiles by road segment, road class, city, day of week, and time window.
  • Detect recurring congestion patterns, abnormal delay, road closure signals, and travel-time instability from GPS probe data.
  • Support construction of piecewise travel-time / speed functions for dynamic routing and simulation.
  • Work with MapOps and Engineering to identify roads where missing restrictions, wrong road class, wrong speed, or bad topology cause routing degradation.
D. Personalized Route Planning & Driver Preference Modeling
  • Analyze actual route choices to infer driver, vehicle, and fleet preferences: fastest route, toll-avoidance, highway preference, safe route, familiar route, low-congestion route, or compliance-first route.
  • Build preference features and clustering models for user groups: private car users, taxi drivers, logistics fleets, bus / coach drivers, EV drivers, and toll-road users.
  • Support alternative-route ranking: not only fastest route, but also safest, most reliable, lowest toll, lowest violation risk, or most familiar.
  • Segment real-world trajectories into meaningful sub-trips and infer intermediate destinations or route-choice intent.
E. Trajectory Mining & Map Quality Detection
  • Mine GPS trajectories to detect map defects: missing roads, wrong turn restrictions, wrong one-way tags, incorrect toll access, bad entrances, wrong road-side access, and routing through prohibited roads.
  • Compare planned route vs actual route to identify systematic driver deviation.
  • Build anomaly detection for map-matching errors, GPS drift, impossible turns, route snapping failure, and inconsistent speed profiles.
  • Generate prioritized MapOps review queues from data signals: high-traffic, high-impact, high-confidence, and high-risk map issues.
F. Experimentation, Benchmarking & Model Evaluation
  • Design experiments with clear baseline, treatment, success metric, guardrail metric, and rollout criterion.
  • Build offline benchmarks for ETA, route ranking, travel-time prediction, route adherence, trajectory segmentation, and map-defect detection.
  • Evaluate model performance using MAE, MAPE, RMSE, precision, recall, F1, calibration, lift, cost reduction, SLA compliance, and operational review cost.
  • Conduct error analysis and identify whether issues come from data sparsity, noisy GPS, map error, routing-engine behavior, model bias, or product UX.
G. Documentation, SOPs & Team Enablement
  • Build documentation for metric definitions, feature specifications, experiment reports, model cards, RCA notes, and benchmark datasets.
  • Translate analytical findings into concrete MapOps, Engineering, Product, or Business actions.
  • Support junior team members in SQL, Python analysis, geospatial data processing, and experiment design.

Requirements

Must-Have

  • Bachelor's degree in Data Science, CS, Applied Mathematics, Operations Research, Statistics, Transportation Engineering, EE, GIS, or related field.
  • 0–2 years in data science, mobility analytics, routing, logistics, map data, optimization, or machine learning.
  • Strong Python skills for data analysis, modeling, and experiment automation.
  • Solid SQL skills; comfortable with large trip tables, GPS logs, routing logs, and geospatial joins.
  • Basic knowledge of ML: regression, classification, clustering, model validation, train/test split, leakage prevention, and error analysis.
  • Basic understanding of graph concepts: nodes, edges, shortest path, route cost, road network, trajectory, and map matching.
  • Ability to work with messy mobility data: noisy GPS, sparse probes, inconsistent timestamps, missing labels, outliers, and biased samples.
  • Strong English reading ability for technical papers and routing / optimization documentation.

Nice-to-Have

  • Knowledge of operations research: LP, mixed-integer programming, VRP, TSP, pickup-and-delivery, time windows, local search, and heuristics.
  • Experience with OR-Tools, PuLP, Pyomo, Gurobi, CPLEX, SCIP, NetworkX, igraph, or similar tools.
  • Experience with geospatial tools: PostgreSQL/PostGIS, QGIS, GeoPandas, Shapely, H3, DuckDB, OSMnx, Valhalla, OSRM, GraphHopper.
  • GPS trajectory processing, map matching, route similarity, Fréchet distance, dynamic time warping, or trajectory clustering.
  • Familiarity with streaming / big data tools: Kafka, Flink, Spark, Airflow, MinIO, or data lake pipelines.
  • GitHub portfolio, Kaggle work, university thesis, optimization project, logistics analytics project, or map/routing notebook.

Soft Skills & Mindset

  • Mathematical and data-driven mindset: able to convert an operational problem into variables, constraints, objectives, and metrics.
  • Skeptical about data quality; does not trust GPS logs, map data, or model output without validation.
  • Comfortable with ambiguity and imperfect real-world systems.
  • Practical orientation: cares about product impact, computation cost, latency, and human review effort.
  • Interested in building Vietnam-specific mobility intelligence rather than only applying generic models.
  • Good balance between research curiosity and production discipline.
