## 1 · Tầm nhìn & selling point (lớp tích hợp)

**Selling point (1 câu):** "Bản đồ TASCO cảnh báo lái xe đúng lúc, cấp độ làn — *chuyển sang làn phải để rẽ*, **tốc độ giới hạn**, **biển báo** — hiện ra **trước** thao tác, không phải sau."

Năng lực này = **mandate chính thức của Mapping Division** ("just-in-time alerts: speed limits, enforcement cameras, turn/one-way restrictions" — trong JD MapOps Navigation). → present như **đang thực thi chiến lược**, không phải đề xuất mới.

| Năng lực tài xế nhận được | Lớp dữ liệu cần | Cách lấy |
|---|---|---|
| ⏱️ Cảnh báo tốc độ | `maxspeed` mọi đoạn | Lớp luật (~$0) + ảnh verify |
| ↪️ Cảnh báo làn/rẽ | số làn + hình học rẽ | Street-view → CNN lane inference |
| 🚸 Cảnh báo biển báo | vị trí + giá trị biển | Street-view + ảnh vệ tinh |

# CÁC MODULE

## [1] Data Enrichment — maxspeed / biển báo / làn  ✅ *(Tuệ — đã điền đủ)*

**Hiện trạng:** đo xong baseline maxspeed toàn quốc (OSM extract 2026-06-06).
- Mạng **tertiary+ = 133,771 km**, chỉ **12.9% có maxspeed** → **thiếu 116,498 km (87.1%)**. Cao tốc gần xong (96.4%); vấn đề ở trunk→tertiary.

**Kế hoạch lấp:**
- **Lớp luật (Quân):** gán candidate maxspeed ~**100%** km thiếu, **$0 ngoài** (suy từ OSM + luật VN).
- **Lớp ảnh (street view):** chỉ verify + xử lý ngoại lệ.
- **Ảnh vệ tinh:** buy-envelope **20,350 km²** (đô thị 15,355 + hành lang đường 5,662 + đảo đất 867, đã trừ trùng) = **~6%** diện tích VN nhờ lọc sparse.
- **Street view:** phủ tertiary+ **~2 tháng** với ~300 tài xế tích cực; ~**15 TB** storage. P1 (trunk+primary, 33k km, ~4 TB) xong trong vài tuần.

**Chi phí / Ask:**
| Khoản | Số | Ghi chú |
|---|--:|---|
| Ảnh vệ tinh | 20,350 km² × **$/km²** | $8 → ~$163K (range $61K–$305K) · **⟵ Vũ chốt giá** |
| Maxspeed review | compute + nhân lực | **⟵ Anh/HR chốt** |
| Street view | ~15 TB + incentive KPI/Loyalty | **$0 mua ngoài** |

> 🔲 [Vũ] điền **$/km²** thực (UP42/Skywatch). 🔲 [Anh/HR] điền **chi phí review + incentive**.

## [2] Traffic & Routing  *(Huy + Vũ)*

**Hiện trạng (theo recording — Huy chủ trì):**
- Engine: **Valhalla**. **Historical traffic speed profile** dựng từ **GPS công khai 1-giây của Mapillary** (tách riêng profile **xe máy vs ô tô**) → nạp vào Valhalla. *(Nguồn này FREE; không phải mua.)*
- **TomTom API = display-only** (overlay đường màu real-time lên base map), **KHÔNG dùng tính ETA**. Caching tile ở backend ~**$8/1000 req**, refresh ~5 phút.
- Display map & traffic map **bắt buộc trùng Way ID**.

**Kế hoạch:**
- **Benchmark hằng ngày ~10,000 OD** vs RAP/Vietmap/Google; chỉ roll-out ETA 1 thành phố khi "**green**".
- Load estimation RPS peak/avg (kinh nghiệm Ahamove/Be) để scale **500K → 2M MAU**; họp SRE/backend (Thứ Tư) về hosting + server load.

> 🔲 [Huy] điền: chi phí TomTom cache thực; trạng thái pipeline Mapillary speed-profile.
> 🔲 [Vũ] điền: doc routing infra + sizing peak/non-peak; chi phí baseline API RAP/Google.

**Chi phí / Ask:** 🔲 — TomTom cache (~$8/1k) + API baseline benchmark + infra routing/hosting (sizing SRE Thứ Tư). *(Recording không nêu con số tổng.)*
